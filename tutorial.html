Make your blog with Python and Pelican for fun and profit

Wed 15 July 2020

What is a static site?

A static website is any site that uses pre-built files. Static sites are ideal for things like blogging, tutorials and simple displays of text. They’re lovely and lightweight to host because they don’t ask the server to execute any code to display them – the server just responds with the files we made earlier. Let’s learn how to make a static blog with Python!

The files that govern the appearance of every site on the web are written in HTML, CSS and Javascript.

To see this background code of a website you can ‘inspect elements’ in your browser.

These are the languages of the web. Your browser knows how to interpret them to display webpages. But they can be hard to work with. I put together a website from scratch in HTML when I was first learning about web development. It took quite a bit of effort to add pages and different fonts, with some Javascript bits and pieces for the interactive parts like buttons. But it still didn’t look very professional – more like it had emerged from the 90s!

So these days, many people prefer to use website generation tools which produce beautiful-looking web pages with less effort and without their having to learn these three languages directly.

Why Pelican?

There are a lot of different static website generation tools out there – this list has over 100!

I use a tool called Pelican which is written in Python and lets me turn Markdown text files into blog posts. I chose this tool because I’m already familiar with Python, and because it’s optimised for what I’m after – making a nice-looking blog with a mixture of images, links and text. It’s open-source and there are a lot of decorative themes available for you to style your blog. Mine uses the Blue Penguin theme. It’s also used by some of the cool kids like Full Stack Python.

Getting started

Pelican works in both Python 2.7 and Python 3. This tutorial will walk you through how to set things up in Python 3 on Ubuntu. It’s a very similar process when using MacOs, and (here is a tutorial)[https://spapas.github.io/2013/10/07/pelican-static-windows/] that explains how to run Pelican on Windows.

First of all, you’ll want to install Pelican. You can do this with the Python package manager pip. Open a command line in the folder you’d like to make your website files in, and type:

pip install pelican
pip install Markdown

Installing the Markdown module is important to support turning your markdown files into web pages. If you don’t have pip yet, you can install it locally by calling curl https://bootstrap.pypa.io/get-pip.py -o get-pip.py.

Next, we call pelican-quickstart. This command sets up all the files and folders you’ll need. It’ll ask you a series of questions. Say ‘Yes’ to creating a Makefile to automate generation and publishing.

(venv)hannah@lovelace:~/Code/pelican-example-site$ pelican-quickstart

Don’t worry, you can change these settings later if you need to.

You’ll see a new set of files and folders has appeared:

content  Makefile  output  pelicanconf.py  publishconf.py  tasks.py

These govern how Pelican generates your website.

  • output is the folder where the final CSS and HTML will appear. It will be empty at the moment.
  • pelicanconf.py is for your settings - that's what you want to alter when you're tweaking the web-pages you display.
  • publishconf.py is the file that deals with published web pages, eg if we want to do anything different when sharing to the web - we don't need to alter that.
  • content is the folder where you put all your Markdown (.md) or reStructuredText (.rst) files for those great blog posts.

Pelican also recognises the folder names 

content
    images
    pages

These are for pictures and extra web-pages you can add later, such as an ‘About Me’ page.

First page

To generate HTML code for our site for the first time and take a look at the results, we call pelican -lr. This means "listen and reload" whenever our contents files change. It gives us a local preview at http://localhost:8000/ - navigate there to see what the site looks like.

The blank example site you'll see

Hurrah!

Aside: Pelican has various commands to generate and preview your HTML files. Their official quickstart tutorial suggests calling two commands, pelican content to generate the HTML output and pelican --listen to preview it in your local browser. I find pelican -lr is more convenient because you can leave it running and it detects any changes you make and automatically updates the output - you just need to refresh your browser. That way you don't need to remember to keep calling pelican content.

Take a look at the output folder and you'll see that we now have some generated files in there, including some CSS and fonts in the theme folder:

(venv)hannah@lovelace:~/Code/pelican-example-site/output$ ls
archives.html  authors.html  categories.html  index.html  tags.html  theme

This isn't personalised yet and it doesn’t have any blog posts. So let’s add one!

Add a blog post

To add a blog post, you just need to make a Markdown file inside the content folder. We’ll call this first-post.md. Note that we need to include a title and some other info in the header.

Title: F1rst P0st!!
Date: 2020-07-15
slug: first-post
Category: hannah’s posts
tags: pelican, first time for everything, new
Summary: Why are we using Pelican for this blog?

This is my first blog post.

# Subheading

Because Pelican is cool.

Nicer looking Markdown

To add images to your posts, you need to add a separate images directory inside the content directory. Linking to these images from a blog post is done like ![Mouseover text]({static}/images/my_image.png)

Things like headings, bullet points, indented quotations and so on can be done in plain Markdown. See the Markdown Cheatsheet for a handy reference.

Themes

There are lots of pre-made themes to style the appearance of your new blog. Take a look at [the Pelican Theme previews](http://www.pelicanthemes.com/] for inspiration. Download one you like from GitHub – the collection is here

Then add a line to your pelicanconf.py file to point at the location you downloaded it to:

THEME = “/home/user/theme-location/theme-name”

(Make sure you use the full path ~/ doesn’t work).

Then call pelican -lr to see your newly themed site! The themes use Jinja2 templates so it's relatively easy to modify them or make your own. My own blog uses the ‘Blue Penguin’ theme.

Hosting your site on the web

So far this website is only being displayed locally on your own computer. It’s time to show it to the world! There are a lot of different options out there for hosting your static website, some free and some paid. A couple of good free ways are GithHub Pages and Fast.io.

Fast.io

Fast.io is a nice way to get started without needing to know other tools. You can point it at a folder in DropBox, Google Drive or Microsoft OneDrive and it’ll refer to the files there for your website. It’s also happy to be pointed at a GitHub repository.

Just make sure wherever you point it contains all of the files that were generated inside your output folder, in the same folder structure.

GitHub Pages

If you already have a GitHub account, GitHub Pages is a great option. It’s free and lets you host a single site named after your GitHub profile – for instance, the address of my GitHub pages blog is https://raspberrycheesecake.github.io/ because my username is RaspberryCheesecake. All you need to do to make one of your own is create a repository called username.github.io where username is your own username, and push the html files generated by Pelican in the ``output``` folder to it.

Something that might trip you up is that you’ll probably want to make two git repositories:

  • one to save your markdown blog posts and settings files like https://github.com/RaspberryCheesecake/pelican-website
  • the other for the HTML output of Pelican driving the website like https://github.com/RaspberryCheesecake/raspberrycheesecake.github.io

Since one is nested inside the other you need to be careful about how you set them up so files aren’t needlessly duplicated. Now I want to make an interactive site!

If you want to go on to make a more interactive web app with things like live data display, feedback forms or online payments, static site builders like Pelican aren’t the right tools for the job. Static sites don’t have a tight coupling between client and server by design, so aren’t responsive enough for these kinds of applications. Why not try using Anvil instead? It allows you to write full-stack web apps with Python.
